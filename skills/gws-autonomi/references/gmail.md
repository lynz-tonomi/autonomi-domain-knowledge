## Sending Patterns

### Simple Send (notifications, alerts)
```bash
gws gmail +send \
  --to "recipient@example.com" \
  --subject "Subject Line" \
  --body "Plain text body"
```

### HTML Send (formatted reports)
```bash
gws gmail +send \
  --to "recipient@example.com" \
  --subject "Weekly QC Summary — Week 12" \
  --body "<h2>Quality Summary</h2><p>Batch pass rate: 97.3%...</p>" \
  --content-type "text/html"
```

### Send with CC/BCC (escalations)
```bash
gws gmail +send \
  --to "qa_lead@tenant.com" \
  --cc "plant_manager@tenant.com" \
  --bcc "compliance_log@tenant.com" \
  --subject "HOLD: Batch 2847 — pH Out of Spec"
```

### Send with Attachment
```bash
# First upload to Drive, then attach
gws drive +upload ./COA_batch_2847.pdf --name "COA-2847"
gws gmail +send --to "supplier@example.com" --subject "COA Request" \
  --body "Please find attached..." --attachment ./COA_batch_2847.pdf
```

## Reading & Triage

### Search Patterns for F&B Operations
```bash
# Supplier emails from last 7 days
gws gmail messages list --params '{"q": "from:@supplier.com after:2026/03/14"}'

# Unread PO confirmations
gws gmail messages list --params '{"q": "subject:(PO OR purchase order) is:unread"}'

# Quality-related with attachments (COAs, lab results)
gws gmail messages list --params '{"q": "subject:(COA OR certificate OR lab result) has:attachment"}'

# BILL.com payment notifications
gws gmail messages list --params '{"q": "from:notifications@bill.com subject:payment"}'
```

### Read Full Message
```bash
gws gmail messages get --params '{"id": "MSG_ID", "format": "full"}'
```

### Automated Triage
```bash
gws gmail +triage
```
The triage command categorizes unread messages and suggests actions. Connect this to agent routing: quality-related emails → Quality Agent, procurement → Procurement Agent, etc.

## Threading & Context

Gmail threads group related messages. When agents reply, they must maintain thread context:

```bash
# Reply maintains thread
gws gmail +reply --message-id MSG_ID --body "Confirmed. Updated delivery to March 25."

# Reply-all for multi-party (supplier + internal team)
gws gmail +reply-all --message-id MSG_ID --body "All parties confirmed. Proceeding with shipment."

# Forward (escalate to different agent/person)
gws gmail +forward --message-id MSG_ID --to "plant_manager@tenant.com" \
  --body "Escalating: supplier cannot meet delivery window. See original below."
```

## Webhooks & Watch

For real-time agent triggers, use Gmail push notifications:

```bash
# Set up watch on inbox
gws gmail +watch --topic "projects/autonomi/topics/gmail-tenant-${TENANT_ID}"

# Watch expires after 7 days — renew automatically
gws gmail users watch --params '{"userId": "me"}' \
  --json '{"topicName": "projects/autonomi/topics/gmail-tenant-${TENANT_ID}", "labelIds": ["INBOX"]}'
```

**Webhook → Agent routing logic:**
1. Gmail notification arrives at Pub/Sub topic
2. `gmail_webhook_processor.py` parses the notification
3. Based on sender/subject/labels, routes to appropriate Autonomi agent
4. Agent processes message and takes action (reply, create record, trigger workflow)

```
Subject: PO #{po_number} — {ingredient_name} | {tenant_name}
Body:
Purchase Order #{po_number}

Supplier: {supplier_name}
Item: {ingredient_name}
Quantity: {quantity} {unit}
Unit Price: ${price_per_unit}
Total: ${total_amount}
Delivery Window: {delivery_start} to {delivery_end}
Ship To: {warehouse_address}

Lot/batch traceability required per FSMA 204.
Please confirm acceptance by replying to this email.

Generated by Autonomi Procurement Agent
```

### Quality Agent — Hold Notice
```
Subject: ⚠️ QUALITY HOLD — Batch {lot_number} | {product_name}
Body:
QUALITY HOLD NOTICE

Batch: {lot_number}
Product: {product_name}
Test Type: {test_type}
Result: {result} (Spec: {spec_range})
Hold Initiated: {timestamp}
Initiated By: Quality Agent (automated)

ACTION REQUIRED: Review hold within 24 hours.
QC inspection scheduled: {calendar_link}

Do not ship or further process this batch until hold is resolved.
```

```
Amount: ${amount}
Method: {payment_method} (via BILL.com)
Reference: {transaction_id}
Supplier: {supplier_name}

This payment was auto-approved per spend policy (threshold: ${threshold}).

```
