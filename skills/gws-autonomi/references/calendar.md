## Event Operations

### View Agenda
```bash
gws calendar +agenda
```

### Create Event
```bash
gws calendar +insert \
  --title "QC Inspection - Lot 2847 Mango Puree" \
  --start-time "2026-03-22T09:00:00-05:00" \
  --end-time "2026-03-22T10:00:00-05:00" \
  --description "Auto-scheduled by Quality Agent. pH reading 3.2 (spec: 3.5-4.0)."
```

### Create Event with Meet Link
```bash
gws calendar events insert \
  --params '{"calendarId": "primary", "conferenceDataVersion": 1}' \
  --json '{
    "summary": "Supplier QBR - Acme Ingredients",
    "start": {"dateTime": "2026-03-25T14:00:00-05:00"},
    "end": {"dateTime": "2026-03-25T15:00:00-05:00"},
    "attendees": [
      {"email": "procurement@tenant.com"},
      {"email": "sales@supplier.com"}
    ],
    "conferenceData": {
      "createRequest": {"requestId": "autonomi-sqbr-2026q1"}
    },
    "description": "Quarterly business review. Agenda: delivery performance, pricing, Q2 forecast."
  }'
```

### Update Event
```bash
gws calendar events patch \
  --params '{"calendarId": "primary", "eventId": "EVT_ID"}' \
  --json '{"summary": "UPDATED: QC Inspection - Lot 2847 (Rescheduled)"}'
```

### Delete Event
```bash
gws calendar events delete --params '{"calendarId": "primary", "eventId": "EVT_ID"}'
```

## Agent-Triggered Events

| Trigger | Agent | Calendar Event | Attendees |
|---------|-------|---------------|-----------|
| Ingredient delivery ETA confirmed | Procurement Agent | "Receiving & QC: {ingredient} from {supplier}" (30 min) | Receiving dock, QC team |
| Batch quality hold issued | Quality Agent | "HOLD REVIEW: Batch {lot}" (60 min, urgent) | QA lead, plant manager |
| Production run scheduled | Production Agent | "Production: {product} — Line {line}" (duration = run time) | Line operator, supervisor |
| Supplier contract expires in 30 days | Procurement Agent | "Contract Renewal Review: {supplier}" (60 min) | Procurement, finance |
| BILL.com payment due in 3 days | AP Automation Agent | "Payment Due: PO #{po} — ${amount}" (15 min reminder) | AP clerk, finance manager |
| FSMA mock recall due (quarterly) | Compliance Agent | "Mock Recall Drill" (120 min) | QA team, plant manager, compliance |
| Maintenance window scheduled | Maintenance Agent | "Planned Maintenance: {equipment}" (duration varies) | Maintenance crew, production |
| New employee food safety training due | HR Agent | "Food Safety Training: {employee}" (120 min) | Employee, trainer |

## Recurring Events

```bash
# Weekly production planning meeting
gws calendar events insert --params '{"calendarId": "primary"}' --json '{
  "summary": "Weekly Production Planning",
  "start": {"dateTime": "2026-03-23T08:00:00-05:00", "timeZone": "America/Chicago"},
  "end": {"dateTime": "2026-03-23T09:00:00-05:00", "timeZone": "America/Chicago"},
  "recurrence": ["RRULE:FREQ=WEEKLY;BYDAY=MO"],
  "description": "Auto-managed by Autonomi. Agenda auto-populated from production schedule."
}'

# Monthly supplier review
gws calendar events insert --params '{"calendarId": "primary"}' --json '{
  "summary": "Monthly Supplier Performance Review",
  "recurrence": ["RRULE:FREQ=MONTHLY;BYDAY=1FR"],
  "description": "Supplier scorecards auto-generated by Procurement Agent."
}'
```

## Timezone Handling

F&B operations often span multiple timezones (supplier in one tz, warehouse in another, HQ in a third). Always use explicit timezone in event creation:

```bash
# Always specify timezone — never rely on default
"start": {"dateTime": "2026-03-22T09:00:00-05:00", "timeZone": "America/Chicago"}

# For cross-timezone coordination, use UTC internally
"start": {"dateTime": "2026-03-22T14:00:00Z", "timeZone": "UTC"}
```

Tenant timezone is stored in `tenants.settings.timezone` — use this as default for all calendar operations.
